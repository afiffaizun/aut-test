# Auth Service Documentation

## Project Overview
Authentication microservice built with Golang using Clean Architecture. Handles user registration, login, JWT & refresh token management, and endpoint protection.

## Technology Stack
- **Language**: Golang
- **Web Framework**: Gin
- **Database**: PostgreSQL (with SQLC for type-safe queries)
- **Cache/Session Store**: Redis
- **Configuration**: godotenv

## Stage 2: Domain Layer ✅ COMPLETED
**Commit**: `feat: add domain entities and repository interfaces`

### Files Created
```
internal/domain/
├── user.go         # User entity and DTOs
├── token.go        # Token entities (AccessToken, RefreshToken, TokenPair)
├── session.go      # Session entity
├── repository.go  # Repository interfaces (contracts)
└── errors.go      # Domain-specific errors
```

### Entities
- **User**: id, email, password_hash, created_at, updated_at
- **AccessToken**: user_id, token_id, expires_at
- **RefreshToken**: id, user_id, token_hash, expires_at, created_at
- **Session**: id, user_id, refresh_token_id, user_agent, ip_address, created_at, expires_at, is_revoked

### Repository Interfaces
- **UserRepository**: Create, GetByEmail, GetByID
- **TokenRepository**: StoreRefreshToken, GetRefreshToken, RevokeRefreshToken, RevokeAllUserTokens
- **SessionRepository**: Create, GetByID, Revoke, RevokeAllForUser

### Domain Errors
- ErrUserNotFound
- ErrInvalidCredentials
- ErrEmailExists
- ErrInvalidToken
- ErrTokenExpired
- ErrTokenRevoked

- idx_refresh_tokens_user_id

### Testing
Prerequisites: PostgreSQL running
```bash
# Create database
docker exec -i exporthub_postgres psql -U exporthub -d exporthub_db -c "CREATE DATABASE auth_service;"

# Apply schema
docker exec -i exporthub_postgres psql -U exporthub -d auth_service < sql/schema.sql

# Test connection
go run cmd/server/main.go
# Expected: "Database connected successfully!"
```

## CI/CD Pipeline

### Workflows

| Workflow | Trigger | Description |
|----------|---------|-------------|
| `ci.yml` | Push/PR to main | Build, Test, Lint, Security Scan |
| `deploy.yml` | Manual dispatch | Build Docker image & Deploy |

### CI Pipeline (ci.yml)

**Jobs:**

| Job | Description |
|-----|-------------|
| lint | golangci-lint, go vet |
| test | go test -v -race ./... |
| build | Build binary |
| docker-build | Docker build + Trivy vulnerability scan |

### Deploy Pipeline (deploy.yml)

**Manual Inputs:**

| Input | Type | Description |
|-------|------|-------------|
| environment | choice | staging or production |
| skip_migrations | boolean | Skip database migrations |

**Jobs:**

| Job | Description |
|-----|-------------|
| build-push | Build & push Docker image to GHCR |
| deploy | Pull image & restart container |
| migrations | Generate SQLC & apply schema |

### GitHub Actions Required Secrets

| Secret | Description |
|--------|-------------|
| GITHUB_TOKEN | Auto-provided by GitHub |
| (Optional) Registry credentials | For private container registry |

### Deployment

```bash
# Manual trigger via GitHub CLI
gh workflow run deploy.yml -f environment=staging

# Or via GitHub UI
# 1. Go to Actions > Deploy
# 2. Click "Run workflow"
# 3. Select environment
```

### Docker Build

```bash
# Build locally
docker build -t auth-service:latest .

# Run with docker-compose
docker compose -f production-compose.yaml up -d

# Or manually
docker run -d -p 8080:8080 \
  --env-file .env \
  ghcr.io/mafif/auth-test:latest
```

| POST | /api/v1/auth/logout | Logout user |
| POST | /api/v1/auth/refresh | Refresh access token |
| GET | /api/v1/auth/me | Get current user |
